## 📊 Майстер-шпаргалка Big-O

### Структури даних (середній / гірший випадок)
| Структура | Доступ | Пошук | Вставка | Видалення | Пам'ять |
|---|---|---|---|---|---|
| **list** (динамічний масив) | O(1) | O(n) | O(1)* в кінець / O(n) | O(n) | O(n) |
| **dict / set** (хеш-таблиця) | — | O(1) / O(n) | O(1) | O(1) | O(n) |
| **deque** (двобічна черга) | O(n) | O(n) | **O(1) з обох кінців** | O(1) кінці | O(n) |
| **зв'язаний список** | O(n) | O(n) | O(1)** | O(1)** | O(n) |
| **heap** (купа) | O(1) min | O(n) | O(log n) | O(log n) | O(n) |
| **BST збалансоване** | O(log n) | O(log n) | O(log n) | O(log n) | O(n) |
| **BST незбалансоване** | O(n) | O(n) | O(n) | O(n) | O(n) |

`*` амортизовано. `**` якщо вузол уже знайдено.

### Сортування
| Алгоритм | Best | Average | Worst | Пам'ять | Stable |
|---|---|---|---|---|---|
| Bubble / Insertion | O(n) | O(n²) | O(n²) | O(1) | ✅ |
| Selection | O(n²) | O(n²) | O(n²) | O(1) | ❌ |
| Merge | O(n log n) | O(n log n) | O(n log n) | O(n) | ✅ |
| Quick | O(n log n) | O(n log n) | O(n²) | O(log n) | ❌ |
| Heap | O(n log n) | O(n log n) | O(n log n) | O(1) | ❌ |
| **Timsort** (Python) | O(n) | O(n log n) | O(n log n) | O(n) | ✅ |
| Counting / Radix | O(n+k) | O(n+k) | O(n+k) | O(n+k) | ✅ |

### Графові алгоритми
| Алгоритм | Складність | Призначення |
|---|---|---|
| BFS / DFS | O(V + E) | обхід, найкоротший шлях (незважений) |
| Topological Sort | O(V + E) | порядок залежностей (DAG) |
| Dijkstra | O((V+E) log V) | найкоротший шлях (ваги ≥ 0) |
| Bellman-Ford | O(V·E) | найкоротший шлях (від'ємні ваги) |
| Floyd-Warshall | O(V³) | усі пари найкоротших шляхів |
| Prim / Kruskal (MST) | O(E log V) | мінімальне кістякове дерево |
| Union-Find | ~O(α(n)) ≈ O(1) | компоненти зв'язності |

## 🎯 Гайд розпізнавання патернів (золото для інтерв'ю)

**«Якщо бачиш X → думай Y»:**

| Сигнал у задачі | Патерн / структура |
|---|---|
| «відсортований масив» + пошук | **Binary Search** (O(log n)) |
| «top-K», «K найбільших/менших» | **Heap** (купа) |
| «усі комбінації/перестановки/підмножини» | **Backtracking** |
| «найдовший/найкоротший підрядок з умовою» | **Sliding Window** |
| «пара/трійка з сумою» у відсортованому | **Two Pointers** |
| «чи є цикл», «знайти середину» списку | **Fast & Slow Pointers** (Floyd) |
| «найкоротший шлях», «рівні», «найближче» | **BFS** |
| «усі шляхи», «зв'язність», «острови» | **DFS** |
| «залежності», «порядок», «розклад» (DAG) | **Topological Sort** |
| «найкоротший шлях зі вагами» | **Dijkstra** |
| «максимум/мінімум», «скількома способами», «оптимально» з перекриттям | **Dynamic Programming** |
| «локально оптимальний вибір» | **Greedy** |
| «чи бачив раніше», «дублікати», «частота» | **Hash (dict/set)** |
| «вкладені дужки», «undo», «найближчий більший» | **Stack** |
| «діапазон у відсортованому», «вставити позицію» | **bisect** (lower/upper bound) |
| «префікси слів», «автодоповнення» | **Trie** |

**Як підходити до будь-якої задачі (фреймворк):**
1. **Уточни** вхід/вихід, межі (n до 10? до 10⁹?), краї (порожній, один елемент, дублікати).
2. **Brute force** спочатку — назви наївне рішення та його Big-O.
3. **Знайди патерн** (таблиця вище) → структура даних, що пасує.
4. **Оптимізуй** — обміняй час на пам'ять (хеш/кеш), або відсортуй.
5. **Код** — чистий, з типами.
6. **Перевір** на краях + оціни фінальну складність (час + пам'ять).

**Орієнтир за межами `n` (яку складність шукати):**
| n до | Прийнятна складність |
|---|---|
| 10–12 | O(n!), O(2ⁿ) — перебір ок |
| ~20 | O(2ⁿ) |
| ~500 | O(n³) |
| ~5000 | O(n²) |
| 10⁶ | O(n log n) |
| 10⁸+ | O(n), O(log n), O(1) |

## 🛠️ Як вчитися за цим курсом

1. **Читай по порядку** 01 → 14 (є залежності).
2. **Запускай код** — копіюй у файл, додавай `print`, дивись поведінку:
   ```bash
   uv run python файл.py    # або просто python
   ```
3. **Перепиши з пам'яті** — справжнє розуміння = відтворити без підглядання.
4. **Розв'язуй задачі** після кожної теми (LeetCode patterns у кінці файлів).
5. **Промовляй вголос** Big-O кожної операції — на інтерв'ю це найважливіше.

> ⚠️ Складності в шпаргалках — стандартні (CLRS), але реальна швидкість залежить від констант, кеша CPU, реалізації. Big-O — про **масштабованість**, не абсолютну швидкість (див. [01_complexity.md](01_complexity.md)).
